Теоретическая часть (Анализ и восстановление данных)

Стоит задача проанализировать синтетический датасет, смоделировать потерю данных и исследовать эффективность методов их восстановления.

4.1. Проблема пропущенных данных и методы восстановления.
В реальных наборах данных часто отсутствуют значения из-за сбоев оборудования, ошибок ввода или недоступности информации. Для продолжения анализа необходимо заполнить эти пробелы.
Методы импутации (восстановления):
* Hot-Deck (Метод горячего реципиента) — заполнение пропусков значениями из других, похожих записей (соседей) в том же датасете. Метод сохраняет распределение данных, но может копировать шумы.
* Сплайн-интерполяция — математический метод, при котором пропущенные значения восстанавливаются с помощью кусочно-заданных функций (сплайнов), обеспечивающих гладкость кривой данных.

4.2. Кластеризация и отбор признаков.
Для выявления скрытых структур в данных используются методы обучения без учителя.
* Алгоритм FOREL — метод кластеризации, основанный на идее объединения точек в сферы фиксированного радиуса. Центр сферы итеративно смещается в центр тяжести точек, попавших внутрь, пока не стабилизируется.
* Отбор признаков (Feature Selection) — процесс выбора наиболее значимых атрибутов датасета. Использование алгоритма SPA (Stochastic Point Adaptation) позволяет найти подмножество признаков, максимизирующих качество разбиения на кластеры (метрика Silhouette), отсеивая шум.

4.3. Оценка качества сравнения кластеризаций.
* Индекс Phi (Phi Index) — мера ассоциации между двумя разбиениями данных (например, исходным и восстановленным). Он вычисляется на основе статистики хи-квадрат ($\chi^2$) и нормализуется размером выборки. Значение индекса показывает степень статистической зависимости между метками кластеров: чем выше значение, тем точнее восстановленная структура данных повторяет исходную.

Основные шаги программы анализа (lab1_analysis.py)

1. Запуск программы и загрузка данных.
Скрипт принимает размер датасета, загружает соответствующий CSV-файл и проводит препроцессинг (обработка нулей, кодирование категориальных признаков).

2. Первичный статистический анализ.
* Расчет описательных статистик (среднее, медиана, мода) для каждого признака.
* Построение гистограмм распределения для визуальной оценки структуры данных.

3. Отбор признаков и настройка кластеризации (на исходных данных).
* Запуск алгоритма SPA для определения "лучших признаков" — тех колонок, которые дают наиболее четкое разделение объектов.
* Динамический подбор радиуса для алгоритма FOREL: программа итеративно уменьшает радиус поиска, пока не будет найдено оптимальное количество кластеров (более 2).
* Кластеризация исходного датасета с оптимальными параметрами и сохранение эталонной визуализации (PCA-проекция).

4. Моделирование потери данных.
Искусственное удаление данных блоками (chunks) в заданных процентных соотношениях (например, 1%, 5%, 10%, 20% пропусков) для проверки устойчивости методов восстановления.

5. Восстановление данных.
* Применение метода Hot-Deck для заполнения пропусков.
* Применение сплайн-интерполяции для заполнения пропусков.
* Сохранение восстановленных датасетов в отдельные файлы.

6. Оценка качества восстановления.
* Вычисление средней относительной ошибки (Mean Relative Error) для числовых признаков путем сравнения восстановленных значений с исходными.
* Повторная кластеризация восстановленных данных с использованием тех же признаков и радиуса, что и для исходного набора.
* Выравнивание кластеров: сопоставление меток кластеров восстановленных данных с эталонными (по максимальному перекрытию) для корректного визуального сравнения.

7. Визуализация результатов.
Генерация сравнительных графиков, показывающих, как методы восстановления исказили структуру данных относительно эталона.

8. Итоговая проверка структурного сходства.
Вычисление индекса Phi (Phi Index) — меры ассоциации между кластерами исходного и восстановленного датасетов. Это позволяет дать количественную оценку того, насколько точно удалось восстановить внутреннюю структуру данных, а не только отдельные значения.
